SMElevate V2 – Loan Request Form Fields (End User)
====================================================
Source: SchemeFormFields table (live database, 2026-07-15)
Form is dynamic — fields are loaded per Financing Scheme selected.

Fields are listed in the order they appear on screen (Section Order → Display Order).
R = Required   O = Optional

HIDDEN / SYSTEM FIELDS (posted with the form, not shown to user)
──────────────────────────────────────────────────────────────────────
SchemeId        — ID of the selected Financing Scheme
SchemeFormId    — ID of the active published form version
FieldValuesJson — JSON array of { fieldName, fieldValue } for all dynamic fields

──────────────────────────────────────────────────────────────────────
NOTES
──────────────────────────────────────────────────────────────────────
- Fields are configurable in Admin → Scheme Forms. The above reflects
  the default seeded form for the standard scheme.
- Conditional fields (HasConditionalVisibility = true) are hidden until
  a trigger field matches the ConditionalExpression (e.g. "FieldName = Value").
- bank_select fields post as AssignedBankId (integer FK to Banks table).
- Dynamic field values are stored in LoanRequestFieldValues table.
  Core LoanRequest columns (NameOfBusiness, Amount, etc.) are also
  mapped directly from FieldName by the controller.
